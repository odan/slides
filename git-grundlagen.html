Git Grundlagen

Einfach erklärt

1. Was ist Git eigentlich?

Git ist wie eine Zeitmaschine für deine Dateien.

  • ✓ Merkt sich jede Änderung
  • ✓ Lässt dich jederzeit zurückgehen
  • ✓ Ermöglicht Teamarbeit ohne Chaos

"Ohne Git ist Programmieren wie Autofahren ohne Sicherheitsgurt."

2. Die wichtigsten Begriffe

📸

Commit

Ein Schnappschuss deiner aktuellen Dateien. Wie ein "Speichern +" mit Beschreibung.

🌳

Branch

Ein separater Arbeitszweig. Du kannst experimentieren, ohne die Hauptversion kaputt zu machen.

3. Neues Projekt starten

Gehe in deinen Projektordner und führe aus:

git init

→ Git erstellt einen versteckten Ordner .git

4. Änderungen speichern

1. Dateien vormerken (Staging):

git add .

2. Commit erstellen:

git commit -m "Meine tolle Änderung"

5. Projekt herunterladen (Clone)

git clone https://github.com/benutzer/projekt.git

Lädt das komplette Projekt inklusive Historie auf deinen Rechner.

6. Änderungen hochladen

git push

Lädt deine lokalen Commits auf den Server (z. B. GitHub).

7. Änderungen herunterladen

git pull

Holt die neuesten Änderungen von anderen Teammitgliedern.

8. Branches – Parallel arbeiten

git branch feature-neue-seite
git checkout feature-neue-seite

Oder kurz:

git checkout -b feature-neue-seite

main ← Hauptzweig

↓

feature-neue-seite ← Dein Feature

9. Typischer Workflow

  1. 1 git pull auf main
  2. 2 Neuen Feature-Branch erstellen
  3. 3 Arbeiten & Commits machen
  4. 4 Branch pushen & Pull Request
  5. 5 Nach Review mergen

Zusammenfassung

Grundbefehle

  • git init
  • git add .
  • git commit -m "..."

Teamarbeit

  • git clone
  • git pull
  • git push

Branches

  • Neuen Zweig erstellen
  • Sicher entwickeln
  • Später mergen

Viel Erfolg mit Git! 🚀
